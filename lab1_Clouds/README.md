# Отчёт по анализу структуры потребления облачных сервисов

### Работа была выполнена нашей командой вместе с Мезенцевым Богданом и Колпаковым Артемом

В ходе выполнения работы был обработан биллинг файл поставщика облачных услуг Amazon Web Services, содержащий данные об использовании сервисов без указания финансовых показателей. Основное внимание уделялось анализу полей: Product Code, Usage Type, LineItemDescription, Operation

На основе этих данных была проведена категоризация потребляемых сервисов согласно иерархической модели:

1. IT Tower - высокоуровневая категория (например, Storage, Compute, Cloud Services).
2. Service Family - группа сервисов (например, Storage & Content Delivery, Artificial Intelligence)
3. Service Type - конкретный сервис (например, Amazon Elastic File System, Amazon GuardDuty)
4. Service Sub Type - подтип услуги (например, Storage Usage, Data Access, Events Analyzed)
5. Service Usage Type - тип потребления (например, Timed Storage, Infrequent Access, Free Events)

В результате анализа выявлены следующие ключевые группы сервисов с возможными аналогами в экосистеме российских облачных платформ:

- Хранилища и управление данными (Amazon Elastic File System, Amazon ElastiCache) - аналоги: Yandex Object Storage, VK Cloud Storage
- Искусственный интеллект и машинное обучение (Amazon Elastic Inference, Amazon Kendra, Amazon Lex) - аналоги: Yandex Cloud AI, SberCloud AI
- Аналитика и обработка данных (Amazon Kinesis, Amazon Redshift) - аналоги: Yandex DataSphere, VK Cloud Data Platform
- Безопасность и идентификация (Amazon GuardDuty) - аналоги: Security Center в российских облаках
- Сетевые сервисы (Amazon Route 53) - аналоги: DNS-сервисы Yandex Cloud и Selectel
- Вычисления и управление ресурсами (Amazon Lightsail) - аналоги: виртуальные машины и контейнеры в инфраструктуре РФ-провайдеров

В результате структуризации данные приведены в аналитический формат, который позволяет чётко определять принадлежность затрат к категориям, агрегировать данные на различных уровнях и проводить управленческий анализ расходов и оптимизировать потребление облачных ресурсов
