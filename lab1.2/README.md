# Отчет по лабораторной 1 со здвездочкой
## Задача
Попробовать взломать nginx любого доступного сайта. Проверить минимум три уязвимости - например path traversal, перебор страниц через ffuf и/или любые другие на ваш выбор.
## Ход работы
В ходе работы были опробованы 3 способа для получения информации с сайта, которая не должна быть доступна обычному пользователю (слово взлом какой-то негативчик дает, поэтому вот такая формулировка) 
1. Path traversal
2. Ffuf
3. http method overdrive

Начали мы с Path traversal, это метод позволяющий получить доступ к файлам за пределами корневой директории сервера путем манипуляции с путями в URL запросах
Вот пример одного из запрсов, которые мы делали:
```curl "https://cocos-lounge.bar//../../../etc/passwd"```

Результат:
403 Forbidden — сервер блокирует такие запросы. В большинстве современных конфигураций nginx такие запросы блокируются, но в устаревших или неправильно настроенных серверах может быть возможен доступ к файлам

Далее Ffuf этот инструмент используется для быстрого перебора возможных путей, файлов или поддоменов на целевом сервере.
Пример запроса:
```ffuf -w Downloads/common.txt -u https://cocos-lounge.bar/FUZZ -mc 200,301,302,403 -t 50 -c -o ffuf_scan.txt"```

Результат это файл, в котором написано, что нам удалось найти, однако все пути также возвращают ошиюку 403.

И на последок мы пробовали http method overdrive это метод, при котором проверяются нестандартные или редко используемые HTTP-методы (например, PUT, DELETE), которые могут быть разрешены на сервере и привести к изменению или удалению данных
Пример запросов:
```curl -X PUT https://cocos-lounge.bar/test.txt -d "test data"``` 

Этот метод также не дал никаких результатов опять ошибка 403, была замечена строчка ```server: ddos-guard```, значит на сервере стоит защита от ddos атак, прикольно))

## Итог
Сайт демонстрирует хороший уровень базовой защиты, против простейших видом атак
